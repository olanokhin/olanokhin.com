// about

I'm Alex — originally from Ukraine, now based in Germany. Before writing a line of AI code I spent 12 years as a live sound engineer — running up to 32 channel consoles at concerts, TV broadcasts, corporate events, and ceremonies. A live show is the only deploy you can't roll back. That shaped how I think about reliability.

The move to AI wasn't a pivot, it was pattern recognition. When I first saw autonomous agent architectures, I recognized the topology: parallel independent processing chains, signal routing, fail-states with no retry button. Same board — different domain.

Right now I'm an AI Engineer at Exyte, building document intelligence pipelines for construction data at scale — multilingual CSVs, OCR on noisy engineering documents, hybrid RAG on Azure. Outside of work I'm doing independent research on agent architectures, specifically how agents should reason about failure, not just succeed.

I'm married to Angelika, we have a 1-year-old daughter Anna. When I'm not shipping, I'm at a live gig somewhere, playing basketball, or at a table tennis table.

// research

BA² Framework — Anokhin²
Biomimetic recursive agent architecture for reliable long-horizon execution. Every node is identical: decompose until deterministically solvable, then crystallize as a validated skill. Graph-based memory eliminates context loss. Dual-channel failure system — Grit (budget) and Frustration (escalation with evidence) — replaces silent loops with structured reasoning about dead ends. Named after P.K. Anokhin, Soviet neuroscientist of Functional Systems Theory.
PoC · Paper in progress
ANV Protocol
Cryptographic nature verification for internet protocols — an "A" layer sitting alongside TLS, MLS, QUIC. Addresses the provability of human vs. automated origin for internet communications.
IETF Draft-00
RCI — Recursive Convergent Inference
Dynamic mixture-of-experts with convergence-gated stopping criteria. Inference halts when output confidence converges rather than at a fixed depth — trading compute for quality adaptively.
Paper + Colab ready
CPAR — Collaborative Peer AI Review
N independent AI reviewers with distinct cognitive profiles conducting blind iterative peer review until consensus. The paper itself was written using this method — 14 iterations with Claude as author and Grok / Gemini / ChatGPT as blind reviewers. All on free tiers, in one evening.
